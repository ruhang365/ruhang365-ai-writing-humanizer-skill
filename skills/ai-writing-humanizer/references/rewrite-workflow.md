# Rewrite Workflow

Use this when the user wants a complete rewrite, not just diagnosis.

## Sequence

1. **Diagnosis first**: list the highest-impact AI-flavor issues. Do not rewrite yet.
2. **Empty sentence pass**: identify sentences that are correct but add no information.
3. **Delete before polish**: remove or merge filler. A useful target is 10-25% compression, but do not delete necessary context.
4. **Concrete replacement**: turn abstract phrases into actions, scenes, examples, tradeoffs, timing, constraints, or decision criteria.
5. **Structure repair**: ensure every section moves the reader forward. Remove recap-only paragraphs.
6. **Voice pass**: keep the author's useful rhythm, attitude, and recurring phrases. Avoid upgrading all language into a generic professional voice.
7. **Platform pass**: adapt only when the destination is known.
8. **Final anti-AI pass**: ask "Where does this still obviously feel like AI?" and fix only residual problems.
9. **Audit contract**: when the rewrite enters a publishing workflow, record the result using `audit-contract.md`.

## Rewrite Priorities

- Preserve stance before improving style.
- Preserve facts before improving flow.
- Preserve useful roughness before making everything smooth.
- Prefer a narrower true claim over a broader unsupported claim.

## Final Anti-AI Pass Checklist

- Does the draft still contain generic transitions?
- Are there correct but empty sentences left?
- Does the author over-explain obvious objections?
- Does it invent reader beliefs?
- Are all paragraphs similar length and shape?
- Did the rewrite erase the author's original attitude?
- Did it add details the user did not provide?
- Did it remove a useful rough edge and replace it with generic professionalism?
- Did the platform version become less native after the rewrite?

## When Not To Rewrite

If the draft is already natural, say so. Offer small edits instead of forcing a full transformation.
